# CalmLog - 「感情の記録から、自分を整える習慣を育てる」

## サービス概要
CalmLogは、日々の感情や気分を手軽に記録し、自分の心の状態を見える化するアプリです。
その記録から「どんな習慣が自分を穏やかに整えてくれるのか」を発見し、少しずつ行動を改善していきます。
感情の記録から、自分を整える習慣を育てる——そんなセルフケアの第一歩をサポートします。

## このサービスへの思い・作りたい理由
日々を過ごす中で、理由もなく気分が沈んだり、もやもやとした感情に包まれることがあります。
その原因を探ろうとしても、仕事や生活習慣、睡眠、気候、人間関係など、要因が複雑に絡み合っていて、自分でもはっきり分からないことが多いものです。

CalmLogは、そんな「自分の心の状態を客観的に見つめることの難しさ」から生まれました。
感情をただ記録するだけでなく、その時の行動や習慣と一緒に振り返ることで、
「自分はどんな行動をすると心が落ち着くのか」「どんな時間が自分を整えてくれるのか」――
そうした“自分なりの穏やかさのパターン”を見つけるきっかけにしたいと考えています。

CalmLogは、心をコントロールするためのツールではなく、
心の波を理解し、受け入れ、少しずつ整えていくためのパートナーのような存在を目指しています。

## ユーザー層について
  メインターゲット

- **穏やかに過ごしたいのに、気持ちの波をコントロールできずに悩む人**
- **日中のすきま時間（1〜2分）で気軽に自分の感情を記録・整理したい人**

  この層を対象にした理由

- 多くの人が「心を整えたい」と感じながらも、時間や余裕のなさから自己管理が後回しになっている。
- 毎日数分の記録でも、感情と行動のパターンを可視化することで、習慣改善やセルフケアのきっかけになる。
- 長文入力ではなく「短時間・低負荷で継続できる設計」が、この層に最もマッチすると考えたため。

## サービスの利用イメージ
- 朝や通勤前に「今日の気分」を3段階で記録
- 日中、もやもやした時に気分とその出来事をサッとメモ
- 夜の振り返りで「落ち着けた瞬間」や「良かった習慣」を記録
- 定期的に、気分の波や整う行動パターンをグラフで確認
- 感情と行動の関係を理解し、自分に合った“整う習慣”を育てていく

## ユーザーの獲得について
- 記録データから「あなたが整いやすい時間・行動パターン」を提案。
- 参加者にツールの使い勝手を体験してもらい、口コミでの拡散を促進。

## サービスの差別化ポイント・推しポイント
###   差別化ポイント

- **「もやもや」と「落ち着き」を分けて記録する二軸構成**
    
    → ネガティブな出来事とポジティブな習慣をそれぞれ観察することで、自己理解から行動変容へ繋がる。
    

この設計により、単なる気分の可視化に留まらず、

自分の整いやすい行動・環境を明確に把握し、日々の習慣改善を促すことが可能になります。

###   推しポイント

- **感情と行動を“二軸で記録”**：心の波と習慣の関係が明確に見える。
- **感情の「原因」と「整い要素」を可視化**：気分の変化をグラフで直感的に把握できる。
- **1〜2分で続けられる“手軽さ”**：短時間入力で無理なく継続できる。
- **「整う習慣」を育てるセルフケア設計**：自分に合った“ご機嫌習慣”を発見し強化できる。
- **落ち着いたデザインと操作感**：穏やかに使い続けられるシンプルなUI。

| ------------ | ------------------------------------- | -------------------------------------------------------- |
| **フレームワーク**  | Ruby on Rails 7.x                     | アプリケーション全体の基盤。Hotwire(Turbo/Stimulus)でスムーズな操作性を実現。       |
| **フロントエンド**  | Tailwind CSS / Stimulus / Chart.js    | シンプルで落ち着いたUI、気分グラフやカレンダー表示を軽量に実装。                        |
| **データベース**   | PostgreSQL                            | 安定した時系列集計・統計処理に適したRDB。Renderでの運用にも相性良。                   |
| **デプロイ**     | Render（または Fly.io）                    | Rails＋Postgresを簡潔に構成できるクラウドPaaS。CI/CDはGitHub Actionsを使用。 |
| **認証**       | Device / OmniAuth Google           | MVPはメール・パスワード認証。将来はGoogleログインを追加予定。                      |
| **外部API／連携** | X(Twitter)シェアAPI ／（将来）AI API(OpenAI等) | MVPではX連携でシェア投稿を実装。将来的に整い習慣レコメンドにAI APIを利用予定。             |
